## Сначала — проверка сложности интерфейса

Запрос крупный (8 пунктов). Чтобы не превратить «Коробочку» в перегруженный планировщик:

- **Единая нижняя панель ввода** (`QuickAddBar`) — одна и та же на главной и на экране таймера. Заменяет текущую кнопку «Новая идея» в таймере. Поле + микрофон + время + «+».
- **Календарь = «История».** Просто переименовать и добавить будущим дням возможность планирования.
- **Повторяемость задаётся в одном месте** (модалка «Добавить задачу»), в шаблоны попадает автоматически.
- **Уведомления по расписанию** — одна простая настройка в сайдбаре.
- Запланированные задачи помечаются подписью «📅 на 2 мая», без новых статусов и фильтров.

Итого добавляется ~3 заметных элемента UI (нижняя панель, дата под заголовком, секция «Уведомления» в сайдбаре). Сложнее не станет — наоборот, в таймере одну кнопку заменим на унифицированную панель.

## План работ

### 1. Фильтр по категории при просмотре задач из карточки дашборда
- В `App.tsx` хранить `tasksFilter: CategoryId[] | null`. Кнопка «список» на карточке дашборда вызывает `setCurrentPage("tasks")` и сохраняет категории секции.
- `TaskListPanel` принимает опциональный `restrictCategories` и фильтрует `source`. Сверху чип «Секция: …» с крестиком для сброса.

### 2. Логика повторяющихся задач — починить
Сейчас `recurring.ts` создаёт задачу один раз при наступлении часа. Нужно: **при отметке выполненной задачи с `templateId` сразу запланировать следующее появление**.
- В точке, где `completed` становится `true`: если у задачи есть `templateId` — рассчитать следующую дату (`daily`/`weekly`/`monthly` + `recurrenceHour`), создать новую задачу с `active=true, completed=false, scheduledFor: timestamp`.
- В `recurring.ts` оставить «страховку»: если приложение не открывали несколько дней — создать пропущенный экземпляр.
- Корректно обновлять `lastCreated`, чтобы не было дублей.

### 3. Периодичность повторения — выпадающий список (мобильный) + повтор в модалке добавления
- В `TemplatesPanel` периодичность (3 кнопки) и дни недели (7 кнопок) на ширине < `sm` заменить на `<select>`. На десктопе оставить кнопки.
- В `AddTaskModal` добавить блок «🔁 Повторять» (свернутый по умолчанию). При включении — `<select>` периодичности + час. При сохранении создаётся `TaskTemplate` (через тот же `saveTemplates`) и сразу первый экземпляр задачи с `templateId`. Шаблон автоматически появляется на странице «Шаблоны».

### 4. «История» → «Календарь»
- В `AppSidebar.NAV_ITEMS` label `История` → `Календарь` (иконка `CalendarDays` уже подходит).
- В `HistoryModal.tsx` заголовок «📊 История и статистика» → «📅 Календарь».
- Тип `PageId` оставить как есть (внутренний идентификатор).

### 5. Планирование задач на будущее через календарь
В «Календаре»:
- Если `selectedDate > today`:
  - Скрываем блок «Активность за день».
  - Заголовок формы → «Добавить задачу».
  - Placeholder → «Новая задача».
  - Скрываем поле «длительность»; время остаётся (час напоминания).
- Сохранение создаёт задачу с `completed=false, active=true, scheduledFor: timestamp`.
- В `Task` добавить `scheduledFor?: number`.
- В `TaskListPanel` запланированные задачи показываем с подписью `📅 запланировано на …`.
- В календарной сетке будущие дни с задачами показывают счётчик другим (приглушённым) цветом.

### 6. Текущая дата под заголовком + стилизация селектов категорий
- Под `🎁 КОРОБОЧКА 5.0` строка: «суббота, 26 апреля» (`Intl.DateTimeFormat('ru', {weekday, day, month})`).
- В `HistoryModal` `<select>` категории заменить на кастомный dropdown с `CategoryIcon`, цветным фоном (`bg-cat-N-bg`) и шрифтом как в `TaskListPanel`.

### 7. Единая нижняя панель ввода (`QuickAddBar`) — главная и таймер
По эскизу: внизу экрана поле «Новая задача…», слева кнопки `09:00` (опц. время), `🔁` (повторение), `🎤` (голос), справа большая зелёная `+`.
- Новый компонент `QuickAddBar` — единый для дашборда и таймера.
- В `Dashboard` рендерится снизу (sticky bottom).
- В `TimerScreen` **заменяет существующую кнопку «Новая идея»** — та же панель, тот же UX. Никакой отдельной модалки на таймере.
- Кнопка `+` / Enter — создаёт задачу. Если время выбрано — задача с `scheduledFor`.
- Микрофон → Web Speech API (`webkitSpeechRecognition`, ru-RU). Распознанный текст пропускается через парсер:
  - регулярки на «на 17:00», «в 10 утра», «завтра», «сегодня», «послезавтра», «в понедельник» → извлекаем дату/время, остальное — текст задачи.
  - Если время найдено — создаём задачу с `scheduledFor`.
  - Если время не названо — небольшой попап «Уточните время» с пресетами (сейчас, 09:00, 12:00, 18:00, своё) + «пропустить».
- Парсер вынести в `src/lib/voiceParse.ts` (юнит-тестируемый).
- Web Speech API недоступен в Firefox — fallback: тост «Голосовой ввод не поддерживается». При желании позже подключим ElevenLabs Scribe через edge-функцию.

### 8. Уведомления-расписание для списка дел на день
- В `localStorage` ключ `daily_reminders`: `{ enabled: bool, times: ["09:00", "18:00"] }`.
- В сайдбаре раздел «Уведомления» расширяется: переключатель + список времён + кнопка «+ время».
- В `App.tsx` `setInterval` (каждую минуту): если `HH:MM` совпадает с расписанным и сегодня в это время ещё не отправляли — формируем уведомление: «Дела на сегодня: N — 1) … 2) … 3) …» (первые 3 активные).
- Звук: системное уведомление + короткий beep через WebAudio при открытом приложении.
- **Ограничение:** браузерные уведомления приходят только если приложение открыто или установлено как PWA с активным Service Worker. Для гарантированных «звонков по утрам» лучше «Установить приложение» (PWA уже настроена). Полноценный Web Push с бэкендом — отдельным шагом, если потребуется.

- `src/App.tsx`: фильтр категорий для TaskListPanel; авто-создание следующего экземпляра при complete; шедулер уведомлений.
- `src/components/QuickAddBar.tsx` (новый): поле + микрофон + время + повтор + «+». Используется и на дашборде, и в таймере.
- `src/components/Dashboard.tsx`: рендер `QuickAddBar` снизу.
- `src/components/TimerScreen.tsx`: убрать кнопку «Новая идея», подключить `QuickAddBar` снизу.
- `src/lib/voiceParse.ts` (новый): извлечение даты/времени из распознанной речи.
- `src/lib/recurring.ts`: рефакторинг — `nextOccurrence(template, fromDate)` + страховка пропущенных.
- `src/components/HistoryModal.tsx`: режим «будущая дата», стилизованный select категорий, заголовок «Календарь».
- `src/components/AddTaskModal.tsx`: блок «Повторять» → создание шаблона.
- `src/components/TemplatesPanel.tsx`: select для recurrence/дня на мобильном.
- `src/components/AppSidebar.tsx`: переименование nav, секция «Расписание уведомлений».
- `src/components/TaskListPanel.tsx`: поддержка `restrictCategories`, бейдж «📅 запланировано».
- `src/lib/notifications.ts`: `scheduleDailyReminders(times, getTasks)`.

**Совместимость данных**
- Старые задачи без `scheduledFor` ведут себя как раньше.
- Существующие шаблоны не ломаются.

**Что осознанно НЕ делаю**
- Серверный Web Push (требует Lovable Cloud + VAPID) — отдельным шагом при необходимости.
- Полноценный NLP — ограничусь регулярками для типовых русских фраз.

После утверждения внесу все изменения и оставлю короткий чеклист проверок.
